# Nexus Gallery - Agentic Context & Project Status

This document is designed to provide immediate context, architecture details, and project status for any Agentic IDE or AI coding assistant working on this repository.

## 🎯 Project Scope
Nexus is a premium, high-performance local media gallery desktop application built to organize, view, and manage images and videos across multiple directories. It emphasizes a visually stunning UI, responsive micro-animations, and robust native OS integrations.

## 🛠 Tech Stack
- **Frontend Framework:** Next.js (App Router, Static Export mode)
- **Desktop Wrapper:** Electron
- **Styling:** Tailwind CSS (Vanilla CSS approach with rich aesthetics, glassmorphism, and dark mode)
- **Database:** SQLite (`better-sqlite3`) for local indexing and rapid querying
- **Media Processing:** 
  - `ffmpeg` / `fluent-ffmpeg` / `ffprobe` (Video streaming, remuxing, seeking, and thumbnail generation)
  - `sharp` (Image thumbnails, metadata extraction, rotation)
- **Icons:** `lucide-react`

## ✨ Features Implemented
- **Directory Management:** Recursive folder scanning and watching. Users can add root folders which populate a dynamic `FolderTree` sidebar.
- **Media Viewer:** 
  - **Images:** Zooming, panning, dragging, 90-degree rotation, and native Windows "Set as Wallpaper".
  - **Videos:** Custom player overlay. Natively plays `.mp4` and `.webm`. 
  - **MKV Streaming:** Uses an internal HTTP server (`main/index.ts`) to dynamically remux (stream copy) `.mkv` files via `ffmpeg` to the frontend without heavy transcoding.
- **Advanced Navigation:** Fully robust keyboard shortcut system (Arrow keys for media navigation, Shift+Arrows for video seeking, F for Fullscreen, Arrow Up/Down for Volume). Includes a custom Toast Notification system for visual feedback.
- **Native OS Integration:** Reveal in Explorer, Copy to Clipboard (using PowerShell on Windows), File Deletion.
- **Tagging System:** Users can add and remove custom tags to media items, stored in the SQLite DB.
- **Premium UX:** Utilizes a custom `InteractiveButton` component everywhere to guarantee consistent scale-down click effects, hover states, and smooth transitions.

## 🚧 Shortcomings & Known Limitations
- **Video Seeking on MKV:** Because the streaming server sends a chunked transfer without Content-Length, HTML5 native byte-range seeking is impossible. Seeking is "faked" by changing the video's `src` URL with a `&startTime=` parameter, which restarts the ffmpeg process. This works but introduces a slight buffering delay.
- **State Management:** Currently, there is no global state manager (like Zustand or Redux). State like `mediaItems` is fetched directly from Electron IPC inside individual pages (`page.tsx`, `settings/page.tsx`). 
- **Sorting/Pagination:** Sorting relies heavily on the DB query. Currently fetching up to 100k items directly into React state for local folder filtering. This could become a performance bottleneck for massive libraries if not properly virtualized.
- **Static Next.js:** Because this is an Electron app, Next.js must be statically exported (`output: 'export'`). This means server-side rendering (SSR), API routes, and Next.js middleware cannot be used. All backend logic MUST live in the Electron `main/` process.

## 🤖 Directives for Agentic IDEs
When making changes to this codebase, adhere to the following rules:
1. **Next.js Constraints:** Do NOT write Next.js API routes (`app/api/...`). Use Electron's IPC (`ipcMain.handle` in `main/index.ts` and `window.electronAPI` in the frontend).
2. **UI/UX Standard:** Ensure all new interactive elements use the `InteractiveButton` component or replicate its interactive states. Do not use generic colors; use curated palettes (zinc, indigo) and respect the glassmorphism theme.
3. **PowerShell Execution:** When executing complex shell commands on Windows (e.g., Clipboard, Wallpaper), ALWAYS use `powershell -EncodedCommand` by converting the script to a Base64 UTF-16LE string. `child_process.exec` with raw strings is brittle and will fail on spaces or special characters.
4. **Video Elements in React:** Remember that React state does NOT automatically sync to native DOM properties on `<video>` elements (like `.volume` or `.currentTime`). You must manually mutate `ref.current.volume` alongside React state.
5. **Layout Structure:** The app uses a full-height flex layout (`h-screen overflow-hidden`). Scrollable areas must be explicitly defined (e.g., `flex-1 overflow-y-auto`).

## 📁 Directory Structure
- `src/app/` - Next.js Pages and Layouts
- `src/components/` - React Components (`MediaViewer.tsx`, `Sidebar.tsx`, etc.)
- `main/` - Electron backend logic (`index.ts`, `db.ts`, `scanner.ts`)
- `thumbnails/` - Automatically generated by `scanner.ts` (stored in AppData in production)
